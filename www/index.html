About Scripturian

Scripturian is a compact library that makes it very easy to integrate scripting languages (which support JSR-223) into your Java-based applications. See the main site for more information, live demos and full documentation.

Features

Scripturian's central feature is that is provides embedded scripting support for many scripting languages, allowing you to mix plain text and embedded "scriptlets," in the style of PHP, JSP and ASP. Any application that generates text can make good use of it, at the minimum as a templating solution, but at the maximum as a fully programmatic way to generate text. For example, a blogging or content management application can use it to give users complete control over theming/skinning.

It's not always easy to configure and start arbitrary scripting engines, contexts, etc. Scripturian does it all for you. For example, any file with <%python at the beginning will start the appropriate script engine. It's a straightforward equivalent of the #!/ used by shells.

Finally, it provides an easy way to bootstrap Java applications using scripting, by delegating the main() method straight into an embedded script. It makes it easy to quickly make changes in how your applications start up without recompiling it.

Supported Scripting Languages

  • JavaScript
  • PHP
  • Python
  • Ruby
  • Groovy

Uses

The Restlet framework has an optional extension that uses Scripturian to allow for implementing REST resources in script, generating HTML with embedded scripts, and bootstrapping the Restlet components. The result is that it's possible to develop very powerful REST applications using only scripting languages, without a single line of Java code, effectively allowing for "Ruby on Restlet", "Python on Restlet", etc.

Please see the Restlet main site for more information.
